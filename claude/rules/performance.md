# Performance (작업 효율/컨텍스트 관리)

이 문서는 `everything-claude-code/rules/performance.md`의 “컨텍스트 관리/계획 모드” 아이디어를 참고해,
Codex CLI 환경에서 실용적으로 적용한 규칙입니다.

---

## 1) 컨텍스트를 아끼는 방식(권장)
- 한 번에 파일을 통째로 붙여 넣지 말고, 필요한 라인/구간만 확인합니다.
- 불필요한 로그/출력을 길게 늘어놓지 않습니다.

---

## 2) 도구 사용 원칙(강제)
- 파일 찾기/검색은 `rg` 우선
- 서로 독립적인 파일 읽기/조회는 병렬 실행(가능하면 `multi_tool_use.parallel` 활용)

---

## 3) 복잡한 작업은 계획부터(강제)
- 여러 파일/여러 단계가 필요한 작업이면, `update_plan`으로 단계/검증 포인트를 먼저 잡습니다.
