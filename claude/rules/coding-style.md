# Coding Style (코드 스타일) - 일관성 규칙

이 문서는 `everything-claude-code/rules/coding-style.md`의 “읽기 쉬운 코드/작은 단위/에러 처리” 원칙을 참고해,
MalgnLMS(JSP+DAO+템플릿, 그리고 project/React)에 맞게 정리했습니다.

---

## 1) 공통(강제)
- 의미 있는 변수/함수명 사용(한 글자 변수 남발 금지)
- 너무 긴 로직은 “상위 메소드=목차, 하위 메소드=구현” 구조로 분리(가능한 범위에서)
- 중첩 if/for는 4단 이상 깊어지지 않게 정리(가능한 범위에서)
- 새로 추가/수정한 로직에는 **한글 주석으로 ‘왜’**를 남깁니다.

---

## 2) TypeScript/React(project)에서(권장)
- 상태/배열/객체는 가능하면 불변(immutable) 패턴으로 처리합니다.
- API 응답은 `rst_data` 형태/타입(배열/단건) 차이를 항상 방어합니다.

---

## 3) JSP/DAO에서(권장)
- “기준값(DB) vs 표시값(JSON/캐시)”이 다를 수 있으니, 기준값을 먼저 확정합니다.
- 입력값은 숫자/문자 포맷을 먼저 정규화하고, 쿼리는 가능하면 바인딩을 사용합니다.
