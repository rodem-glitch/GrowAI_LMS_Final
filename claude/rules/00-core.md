# [강제] 00-core (항상 지켜야 하는 규칙)

왜 이 규칙이 필요한가?
- 실서비스 제품에서는 “작동은 되는데 데이터가 꼬이는” 문제가 가장 위험합니다.
- 그래서 에이전트가 매번 같은 기준으로 확인하고, 최소 변경으로 수정하도록 강제합니다.

---

## A. 절대 금지(강제)
- 문서 삭제 금지: `AGENTS.md`, `CLAUDE.md`, `claude/` 아래 파일은 임의 삭제/이동 금지입니다.
- 무단 대규모 리팩터링 금지: 사용자 요청 없이 파일 이동/광범위 네이밍 변경/구조 재편 금지입니다.
- 근거 없는 추측 수정 금지: 코드/DB 확인 없이 “아마 이거겠지”로 고치지 않습니다.

---

## B. 작업 전 체크리스트(강제)
- [ ] 문제를 1~2문장으로 정리했나요?
- [ ] 관련 화면(JSP) → 템플릿(HTML) → DAO(쿼리) 흐름을 따라가 봤나요?
- [ ] DB 기준값(어느 테이블/컬럼이 진짜인지)을 확인했나요?
- [ ] `site_id`, `status` 조건이 빠지지 않는지 확인했나요?
- [ ] `claude/rules/security.md`의 “하드코딩/권한/입력값” 체크를 확인했나요?

---

## C. 작업 후 체크리스트(강제)
- [ ] `git diff`로 의도치 않은 변경이 없는지 확인했나요?
- [ ] 실제 화면/기능을 최소 1번은 동작 확인했나요?
- [ ] `project/`를 수정했다면 `npm run build`를 실행했나요?
- [ ] 결과 보고에 “문제/원인/해결/영향/검증”이 포함됐나요?
- [ ] 커밋 전 `claude/rules/git-workflow.md`, `claude/rules/testing.md`를 확인했나요?
