# Testing (테스트/검증) - 필수 확인

이 문서는 `everything-claude-code/rules/testing.md`의 “작업 후 반드시 검증” 철학을 참고해,
MalgnLMS에 맞게 현실적으로 적용한 체크리스트입니다.

> 참고: 이 저장소는 레거시(JSP) 비중이 커서 “자동 테스트 80%” 같은 기준을 그대로 강제하기 어렵습니다.
> 대신 **기능 확인 + DB 확인 + 빌드 확인**을 필수로 강제합니다.

---

## 1) 최소 검증(강제)
- [ ] 변경한 화면/기능을 실제로 1회 이상 동작 확인
- [ ] DB가 관련되면, 최소 1개 이상 “조회 SQL”로 저장 결과 확인
- [ ] `project/` 변경이 있으면 `npm run build` 성공 확인

---

## 2) 가능하면 추가(권장)
- `project/`에 테스트가 이미 있다면 해당 테스트 실행
- 회귀 위험이 큰 수정이면(진도/출석/결제 등) “엣지 케이스” 1~2개 추가 확인
