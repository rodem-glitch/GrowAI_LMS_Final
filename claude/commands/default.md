# commands/default (기본 실행 절차)

이 문서는 “작업 유형이 애매할 때” 기본으로 따르는 절차입니다.

1) 관련 파일 찾기
- `rg`로 화면(JSP/HTML)과 API/JSP/DAO를 찾습니다.

2) 기준 데이터 확정
- 화면에 보여야 하는 값이 “어디에 저장되는지(테이블/컬럼)”를 먼저 확정합니다.

3) 최소 변경으로 수정
- 기존 패턴(JSP+DAO+템플릿)을 존중합니다.
- 새 로직에는 한글 주석으로 “왜”를 남깁니다.

4) 검증
- 수정한 화면을 직접 열어 확인합니다.
- DB가 연관되면 조회로 값이 맞는지 확인합니다.

5) 빌드/마무리
- `project/`를 건드렸으면 `npm run build`를 실행합니다.
- 마지막에 `git status`로 변경 파일을 확인합니다.
